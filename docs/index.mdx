# CyberAi

**AI-Powered Smart Contract Security & Orchestration Platform**

## What is CyberAi?

CyberAi is a **GitHub-native control plane** that serves as the central coordination point for a decentralized AI ecosystem. Unlike traditional SaaS platforms, CyberAi is repository-first, leveraging GitHub's infrastructure for deployment, automation, and collaboration.

## Role of This Repository

This repository (`SolanaRemix/CyberAi`) functions as:

### 1. Source of Truth for cyberai.network

- **GitHub Pages Site**: Deployed from the `main` branch via automated workflows
- **Documentation**: Comprehensive guides, architecture docs, and operational runbooks
- **Custom Domain**: cyberai.network with DNS configuration

### 2. Operator Control Plane

- **Contract Registry**: Machine-readable contracts for all ecosystem participants
- **Prompt Library**: Operator-grade instructions for agents
- **Automation Tools**: Bootstrap and audit scripts for ecosystem management
- **CI/CD Workflows**: Automated validation and deployment

### 3. Contract + Prompt Registry

- **Agents**: AI agents for code generation, security audits, and automation
- **Repositories**: Ecosystem repositories like SmartBrain (runners) and GitAntivirus (security)
- **Runners**: Execution environments for agents
- **Prompts**: Detailed instructions that agents consume directly

## No Abstract SaaS

CyberAi is a **GitHub-first, repository-native control plane**:

- No external infrastructure required
- All state in version control
- GitHub Actions for automation
- GitHub Pages for deployment
- GitHub API for integration

## Key Features

### Contract Registry

Machine-readable JSON contracts define:

- Capabilities and actions
- Requirements and dependencies
- Metadata and ownership
- Compliance and security status

All contracts validated against JSON Schema on every commit.

### Prompt Library

Operator-grade prompts provide:

- Clear input/output specifications
- Step-by-step procedures
- Non-goals and constraints
- References to specific directories

Agents consume prompts directly from the repository.

### Automation Tools

Shell scripts for operators:

- **Bootstrap**: Verify environment, install dependencies, validate contracts, build site
- **Audit**: Validate contracts, check workflows, scan for security issues

### Static Site

Astro-based site deployed to GitHub Pages:

- Documentation and guides
- Operator console
- Ecosystem visualization
- Contract browsing

## Workflows

### PR Flow

```
Developer → Push to branch
    ↓
GitHub Actions (ci.yml)
    ↓
Bootstrap & Audit
    ↓
Contract Validation
    ↓
[Pass] → Merge allowed
[Fail] → PR blocked
```

### Deployment Flow

```
Push to main
    ↓
GitHub Actions (pages-deploy.yml)
    ↓
Build site (npm run build)
    ↓
Deploy to GitHub Pages
    ↓
Available at cyberai.network
```

### Contract Discovery Flow

```
SmartBrain Runner
    ↓
GET github.com/.../contracts/agents/
    ↓
Parse contracts
    ↓
Execute agents per contract
```
